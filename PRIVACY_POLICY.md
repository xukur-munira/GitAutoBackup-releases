# Git Auto Backup 隐私政策

最近更新日期：2026 年 4 月 2 日

欢迎使用 Git Auto Backup。我们重视你的个人信息和数据安全。本隐私政策用于说明，在你使用 Git Auto Backup 桌面软件及相关安装程序、自动推送、托盘后台运行、日志记录和远端同步功能时，我们如何收集、使用、存储和保护信息。

如果你不同意本隐私政策的任何内容，请停止安装或使用本软件。

## 1. 本政策适用范围

本政策适用于 Git Auto Backup 桌面软件、安装程序、卸载程序、自动备份功能、托盘常驻功能以及与 Git 仓库同步相关的本地功能。

本政策不适用于以下内容：

- GitHub、Microsoft、Git、Windows 或其他第三方服务自身的隐私做法
- 你在自己的 Git 仓库中主动提交、存储或分发的业务数据
- 由第三方平台单独提供的账号体系、支付体系或云服务

## 2. 我们处理的信息类型

Git Auto Backup 本身不要求你注册账号，也不内建在线用户系统。软件主要处理以下几类信息：

### 2.1 本地配置与运行信息

- 仓库路径
- 分支名称
- 受管范围
- 同步模式
- 自动启动选项
- 托盘状态与任务状态
- 本地日志文件路径与运行日志

### 2.2 Git 相关配置信息

- Git 用户名
- Git 邮箱
- Origin 地址
- SSH Key 文件名或生效密钥信息
- 本地 Git 状态
- 提交状态
- 未推送文件列表

### 2.3 本地文件与仓库元数据

- 文件路径
- 文件修改时间
- 文件大小
- 文件状态（新增、修改、删除、未跟踪）
- 本地与远端内容对比所需的哈希值或对象标识

### 2.4 安装与卸载信息

- 安装目录
- 开始菜单快捷方式
- 桌面快捷方式
- 本地卸载注册表项
- 本地自动启动注册表项

### 2.5 日志与故障信息

- 本地运行日志
- 安装日志
- Git 命令执行错误信息
- 托盘与窗口状态变化记录

## 3. 我们如何收集信息

我们通过以下方式收集信息：

- 你在软件界面中主动填写或选择
- 软件读取本机 Git 配置与本地仓库状态
- 软件在本地执行文件扫描、内容对比和托盘状态更新时自动生成
- 安装程序、卸载程序和自启动配置在本地系统中写入或读取时生成

我们不会在未告知你的情况下额外采集与你使用目的无关的信息。

## 4. 我们如何使用信息

我们使用上述信息仅用于以下目的：

- 实现本地 Git 仓库自动备份与远端同步
- 展示当前仓库状态、分支状态、任务状态与最近推送记录
- 支持托盘控制、暂停、恢复、立即推送与退出
- 支持安装、升级、卸载和快捷方式创建
- 支持检测旧版本安装目录并回填默认路径
- 支持日志记录、故障排查和运行稳定性改进
- 防止将明显不适合自动提交的敏感文件或中间产物误推送到远端

## 5. 我们不会如何使用信息

除非你明确授权或法律法规另有要求，我们不会：

- 将你的 Git 用户名、邮箱、仓库路径或文件列表出售给任何第三方
- 将你的本地仓库内容上传到 GitHub 之外的未知服务器
- 将你的本地文件内容用于广告、画像、推荐或营销
- 在你不知情的情况下把本地敏感文件发送到我们的服务器
- 在后台建立与产品功能无关的远程数据采集通道

## 6. 本地存储与日志

Git Auto Backup 主要在你的本机存储运行数据。常见存储内容包括：

- 本地配置
- 运行状态缓存
- 恢复快照
- 本地日志文件
- 安装目录和快捷方式
- 卸载信息和自动启动信息

这些内容通常保存在你的本地仓库目录、应用目录、开始菜单目录、桌面目录、注册表或系统临时目录中。

日志和恢复文件主要用于排查异常、恢复操作上下文和提示同步状态。你可以根据实际需要删除相关日志或卸载软件。

## 7. 远端同步与第三方平台

当你使用远端同步、自动推送或远端内容对比功能时，软件会与你配置的 Git 远端进行通信，例如 GitHub。

请注意：

- 你推送到 GitHub 的数据，受 GitHub 自身的隐私政策和服务条款约束
- 你使用的 Git 用户身份、邮箱、提交信息和仓库内容，可能会按 Git/GitHub 的标准行为存储在对应远端平台
- 我们无法控制第三方平台如何处理你主动推送上去的数据

因此，建议你同时阅读所使用代码托管平台的隐私政策。

## 8. 敏感信息与排除机制

Git Auto Backup 设计上会尽量减少将敏感文件自动纳入推送范围，例如：

- `.env` 及相关环境配置文件
- `.pid`
- `.pyc`
- 运行时缓存
- 构建中间产物
- 部分证书、密钥和敏感后缀文件

但你仍应自行确认：

- 仓库 `.gitignore` 是否正确
- 受管范围是否设置合理
- 远端推送前是否包含不应提交的业务数据
- 你本地仓库中是否已存在历史敏感内容

本软件提供排除和审查机制，但不应被视为唯一的数据安全保障手段。

## 9. 数据共享与披露

我们不会主动向第三方共享你的本地数据。以下情况除外：

- 为完成你主动发起的 Git 远端同步，必要地向你指定的 Git 托管平台发送数据
- 法律法规、司法机关或监管要求
- 为保护软件、用户或第三方的合法权益，在法律允许范围内进行必要披露

## 10. 数据安全

我们采取以本地控制为主的数据处理方式，并尽量减少非必要的远端依赖。为降低风险，我们建议你：

- 使用受信任的 Git 托管平台
- 妥善管理 SSH Key、访问令牌和 Git 凭据
- 避免把敏感业务数据直接放入代码仓库
- 定期检查 `.gitignore` 和仓库历史
- 在共享设备上谨慎启用自动登录和自动启动

任何软件都无法保证绝对安全。你应结合自己的系统安全策略、访问控制和仓库治理规则共同使用本软件。

## 11. 卸载与删除

你可以随时卸载 Git Auto Backup。卸载后：

- 安装目录会被移除
- 开始菜单和桌面快捷方式会被移除
- 自动启动配置会被删除
- 本地卸载注册表项会被删除

但以下数据可能仍需你手动处理：

- 你本地仓库中的 Git 提交历史
- 已推送到远端仓库的提交记录
- 你手动保留的日志、快照或备份副本
- 其他第三方平台上的历史数据

## 12. 未成年人说明

本软件主要面向具备基本 Git 和本地系统操作能力的用户。若你是未成年人，请在监护人指导下使用本软件。
