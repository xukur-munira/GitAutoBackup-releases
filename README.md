# Git Auto Backup

自动监控文件变化并推送到 GitHub 的智能备份工具。

## 功能特点

- 实时监控文件变化，自动创建 Git 提交
- 智能分块提交，支持大文件和大量文件的备份
- 定时自动备份（可配置间隔时间）
- 启动时自动备份
- 友好的图形界面，实时显示备份进度
- 支持多仓库管理
- 自动处理 Git 冲突和分支同步

## 系统要求

- Windows 10/11 (64位)
- Git 已安装并配置
- 已配置 GitHub SSH 密钥或 HTTPS 认证

## 安装

### 方式 1：使用安装程序（推荐）

1. 下载最新版本的 `GitAutoBackupInstaller.exe`
2. 双击运行安装程序
3. 按照提示完成安装
4. 从开始菜单启动 "Git Auto Backup"





## 使用说明

### 首次配置

1. 启动程序后，点击"添加仓库"
2. 选择要备份的本地 Git 仓库路径
3. 配置备份选项：
   - 启动时自动备份
   - 文件变化时自动备份
   - 定时自动备份（设置间隔时间）
4. 点击"保存配置"

### 手动备份

点击"立即备份"按钮，程序会：

1. 检测所有未提交的文件变化
2. 自动创建提交（如果文件过多会分块提交）
3. 推送到远程仓库

### 自动备份

启用自动备份后，程序会在后台监控文件变化：

- 检测到文件修改后自动提交
- 按设定的时间间隔定时备份
- 启动时执行一次完整备份



## 高级功能

### 分块提交

当检测到大量文件变化时，程序会自动分块提交：

- 每个提交最多包含 100 个文件
- 自动计算最优分块策略
- 避免单次提交过大导致推送失败

### 冲突处理

程序会自动处理常见的 Git 冲突：

- 使用 merge 策略同步远程变化
- 自动清理中断的 rebase 状态
- 智能重试失败的推送操作

## 故障排除

### 推送失败

如果推送失败，检查：

1. 网络连接是否正常
2. GitHub SSH 密钥是否配置正确
3. 是否有权限推送到远程仓库

### 文件监控不工作

1. 确保仓库路径正确
2. 检查是否有文件被 .gitignore 忽略
3. 重启程序重新加载配置
